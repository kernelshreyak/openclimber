# OpenClimber v2.1 Dev Log

## Scope
This iteration focuses on replacing the old ledge-specific character flow with a procedural cube-based climber and making climbing work against generic `ClimbableSurface` objects.

## What Changed
- Replaced the imported character dependency with a procedural `Node3D` rig built from cubes.
- Added a simple cube head, torso, pelvis, shoulders, hands, shoes, hair cap, neck, and face plate so the character reads more clearly in-game.
- Kept the body organized around explicit joints for both arms and legs so each limb can be controlled independently later.
- Generalized climbing around `ClimbableSurface` instead of a hard-coded ledge name.
- Aligned the character to the active surface normal while climbing so angled test surfaces work.
- Added wall traversal while attached, plus jump/drop behavior and a short regrab cooldown after release.
- Added a third-person camera with mouse orbit, pan, zoom, and a `camera_center` action.
- Fixed multiple direction/facing issues between ground movement, climb alignment, camera recentering, and probe direction.
- Tuned the idle pose, climb pose, and limb lengths to reduce clipping and make the silhouette cleaner.

## Notable Fixes During v2.1
- Fixed parse errors in `climbing.gd` by making raycast-related variables explicitly typed.
- Removed the stale `building_1` instance from the main scene when it started producing invalid parent-path warnings.
- Fixed camera flicker by making the camera follow the player after the final physics result each frame.
- Fixed the character starting facing the wrong direction and corrected the climb probe direction to match the current facing convention.

## Current Result
- Character starts in a neutral upright pose.
- Climbing surfaces can be tested at different angles.
- Camera follows the player, supports manual adjustment, and can be centered with `C`.
- The rig is still cube-based, but now reads as a proper low-poly climber instead of a placeholder stack of parts.
